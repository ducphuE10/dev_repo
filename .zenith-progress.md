# Zenith Progress Log

## Codebase Patterns
<!-- Consolidated knowledge from all iterations. Read this FIRST. -->
- The repository currently contains only planning and product docs; the described monorepo structure has not been scaffolded yet.
- The intended stack is TypeScript throughout with pnpm workspaces and Turborepo coordinating `apps/api`, `apps/mobile`, `apps/web`, and shared `packages/*`.
- The workspace namespace is `@dupe-hunt/*`, with root-level `pnpm lint`, `pnpm typecheck`, and `pnpm test` serving as the first stable quality entry points during scaffold iterations.
- Early bootstrap verification lives under `scripts/`, while app-level `dev` commands can stay as lightweight long-running stubs until the framework-specific implementations land.
- Shared object contracts belong in `packages/types`, shared database code belongs in `packages/db`, and apps must not import directly from each other.
- Shared TypeScript defaults now live in `packages/config/typescript/base.json`, with the root `tsconfig.json` extending that base and owning workspace-specific `paths` mappings.
- Environment contracts are centralized in `packages/config/src/index.ts`, while each app keeps a matching `.env.example` file as the source-of-truth template for local setup.
- Fastify API routes are expected to use Zod validation, consistent `{ error: { code, message } }` responses, Drizzle for database access, and BullMQ for heavy async work.
- Mobile screens should stay thin: screen files in `screens/`, reusable UI in `components/`, business logic in custom `hooks/`, React Query for server state, and SecureStore only for auth tokens.
- Next.js web pages are browse-only in v1, prefer server-side data fetching for SEO, and must provide metadata for every public route.
- Product invariants already defined in the docs must hold across implementations: feed results only expose `status = 'active'` posts, affiliate clicks are tracked server-side in the redirect endpoint, and receipt assets remain private.
